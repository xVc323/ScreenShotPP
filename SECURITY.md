# Security Policy

## Supported versions

Security fixes are prioritized for the latest public release and the current `main` branch.

Older preview builds may not receive separate security updates. Please update to the latest release when possible.

## Reporting a vulnerability

Please do not report security vulnerabilities in public issues.

If GitHub private vulnerability reporting is enabled for this repository, use the repository's **Security** tab to create a private vulnerability report.

If private reporting is not available, contact the maintainers privately with enough detail to reproduce and assess the issue.

Helpful reports include:

- A clear description of the vulnerability.
- Steps to reproduce it.
- Affected operating system and ScreenShotPP version or commit.
- Any relevant logs, screenshots, or proof-of-concept details.
- Whether the issue has been shared publicly or with third parties.

Please avoid accessing, modifying, or sharing other people's data while investigating a vulnerability.

## Response expectations

Maintainers will acknowledge valid reports as soon as practical, investigate the impact, and coordinate a fix or disclosure plan when needed.

Security fixes may be released before full technical details are published.
